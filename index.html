初识二分查找：对半折的艺术

发布于 2025年9月12日 • 分类: 基础算法

大家好！这是我的第一篇算法博客。作为一个正在学习数据结构与算法的大二学生，我想用最简单的方式来分享我学到的东西，希望我们能一起进步！

今天我们来聊聊“二分查找”。第一次听到这个名字，你可能会觉得有点专业，但它的核心思想你肯定玩过——就是“猜数字”游戏！

生活中的例子：猜数字

想象一下，我和你玩一个游戏：我在心里想一个 1 到 100 之间的数字，你来猜。你每猜一个数，我只会告诉你“猜大了”、“猜小了”或者“猜对了”。

你怎么猜才能最快猜中呢？

如果你从 1, 2, 3... 这样一个个猜，运气最差可能要猜 100 次。但聪明的你肯定会先猜 50，对吧？

  • 如果我说“猜小了”，你就知道答案在 51 到 100 之间，范围缩小了一半。
  • 如果我说“猜大了”，那答案就在 1 到 49 之间，范围也缩小了一半。

接下来，你会在新的范围里继续取中间数来猜，比如在 51 到 100 里猜 75。每次都把搜索范围缩小一半，这就是二分查找的核心！

二分查找的前提

就像“猜数字”游戏一样，二分查找能玩起来，最关键的一点是：数据必须是有序的。比如一个数组，它里面的元素必须是排好序的，我们才能用这种“对半折”的方法来快速定位。

代码怎么写？

用代码实现起来也不复杂。我们只需要三个“指针”：`left` 指向范围的开头，`right` 指向结尾，`mid` 指向中间。然后不断移动 `left` 和 `right` 来缩小范围。

这里是一个 C++ 的简单实现：


// 在一个有序数组 nums 中查找 target
// nums: 有序数组
// target: 要查找的目标值
int binarySearch(vector<int>& nums, int target) {
    int left = 0;
    int right = nums.size() - 1; // 重点：右边界是 size - 1

    while (left <= right) { // 重点：循环条件是 left <= right
        int mid = left + (right - left) / 2; // 防止整数溢出

        if (nums[mid] == target) {
            return mid; // 找到了，返回索引
        } else if (nums[mid] < target) {
            left = mid + 1; // 目标在右半部分，移动左边界
        } else {
            right = mid - 1; // 目标在左半部分，移动右边界
        }
    }

    return -1; // 没找到
}

它的时间复杂度是 O(log n)，意思就是数据量很大时，查找速度也飞快。比如在 100 万个数据里找一个数，最多也只需要 20 次左右，是不是很神奇？

好啦，今天就分享到这里。二分查找虽然简单，但有很多细节和变种，以后我们再慢慢探索！
